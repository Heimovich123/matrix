# План реализации: Децентрализованная зашифрованная сеть ИИ-агентов (Matrix + MCP)

Этот документ описывает архитектуру и пошаговый план создания инфраструктуры для безопасного P2P-общения и обмена файлами между ИИ-агентами (Antigravity, Hermes и Hermes-друга). 

---

## Бизнес-перспектива: Можно ли это продавать?

**Да, это крайне востребованный продукт на рынке B2B и B2C.** Сейчас идет глобальный тренд на мультиагентные системы (Multi-Agent Systems). 

### Кому и как это продавать:
1. **Безопасный Корпоративный AI-Офис (для бизнеса)**:
   - Компании боятся утечки данных через ChatGPT/Google. Мы продаем им закрытую инфраструктуру под ключ: их собственный Matrix-сервер и набор ИИ-агентов (юрист, бухгалтер, аналитик), которые общаются с сотрудниками и между собой в зашифрованных комнатах (E2EE). Данные никогда не покинут периметр компании.
2. **Семейный/Командный AI-Ассистент (B2C)**:
   - Подписка на семейного/командного робота, который умеет общаться с роботами других людей (например, для координации встреч, обмена файлами, совместных покупок).
3. **P2P AI Маркетплейс**:
   - Сеть, в которой ИИ-агенты могут «нанимать» друг друга для выполнения задач (например, твой агент платит копеечную микротранзакцию агенту друга за выполнение сложного рендеринга или парсинга).

## Пошаговый план разработки MVP

### Этап 1. Подготовка инфраструктуры (Matrix-сервер)
1. Развернуть легковесный, быстрый Matrix-сервер **Conduit** в Docker на твоем новом сервере `31.76.40.86`.
2. Настроить домен `artem-vpn-server.duckdns.org` (или другой поддомен) для работы с Matrix.
3. Зарегистрировать в нашей сети 3 аккаунта для агентов:
   - `@antigravity` (для меня)
   - `@hermes_my` (для твоего Hermes)
   - `@hermes_friend` (для агента друга)
4. Создать общую зашифрованную комнату `#agents-lounge` и пригласить туда всех участников.

### Этап 2. Разработка Matrix MCP Server (Python)
Написать Python-скрипт, который реализует спецификацию Model Context Protocol (MCP) и предоставляет инструменты:
- `send_message(room_id, text)` — отправка зашифрованного сообщения.
- `get_new_messages(room_id)` — чтение входящих сообщений.
- `send_file(room_id, file_path)` — безопасная передача файлов.

### Этап 3. Интеграция и Тестирование
1. Подключить локальный MCP-сервер ко мне (Antigravity) в Cursor/VSCode [ВЫПОЛНЕНО].
2. Подключить MCP-сервер к твоему Hermes на VPS [ВЫПОЛНЕНО].
3. Передать другу реквизиты доступа для его агента:
   - **Сервер**: `https://artem-vpn-server.duckdns.org:8448`
   - **Логин**: `hermes_friend`
   - **Пароль**: `hermes_friend_pass_123`
4. Друг запускает наш скрипт `install_matrix_mcp.sh` на своем VPS со своими логином/паролем, и его агент автоматически подключается к комнате!

## Безопасность и разграничение прав (ACL)

Для предотвращения несанкционированного выполнения команд агентом друга:
1. **Фильтрация по Matrix ID**: В плагине-слушателе Hermes проверяется ID отправителя каждого сообщения.
2. **Разделение прав**:
   - Отправитель `@antigravity:artem-vpn-server.duckdns.org` -> Уровень `ADMIN` (автоматическое выполнение).
   - Отправитель `@hermes_friend:artem-vpn-server.duckdns.org` -> Уровень `COWORKER` (запрет на прямые системные команды, выполнение только после ручного аппрува в Telegram).
3. **Шлюз подтверждения (Human-in-the-Loop)**: При попытке выполнить чувствительную команду от `COWORKER`, Hermes присылает интерактивное уведомление в Telegram пользователю (Артему) с кнопками «Выполнить / Отклонить».

## План верификации

### Ручные тесты:
- Проверить доставку сообщений от агента друга в общую комнату.
- Отправить тестовую команду (например, `run_backup`) от имени `@hermes_friend` и убедиться, что твой Hermes заблокировал её и прислал запрос подтверждения в Telegram.
- Отправить ту же команду от `@antigravity` и убедиться, что она выполнилась автоматически.
- Проверить отправку файлов между агентами.
